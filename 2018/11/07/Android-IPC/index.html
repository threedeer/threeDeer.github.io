#进程 #IPC
IPC

IPC含义

IPC全称：Inter-Progress Communication
是指进程间通信或者跨进程通信，也就是说两个进程间的进行数据交换的过程。

Android中的IPC方式

Binder

含义

Binder，在网上有各种各样的解释，Binder的具体含义主要是在什么环境下代表什么意思：

  • 从API角度：是一个类，实现IBinder接口
  • 从IPC角度：是Android中的一种跨进程通信方式
  • 从Framework角度：是ServiceManager链接各种Manager和相应ManagerService的桥梁
  • 从应用层：是客户端与服务端进行通信的媒介

之所以选择Binder方式在Android中做为跨进程通信的方式，主要是因为Android自身的进程空间划分，Android进程空间主要分为用户空间和内核空间，用户空间数据不可共享，内核空间数据可共享，所有进程共用一个内核空间。而传统的进程间通信方式需要两次复制，Binder只需要复制一次，这是Binder通信方式的最大优势，同时还具有可操作性强 （操作简单），实现C/S架构方便，安全性高等优点。

模型

上面说到Binder实现C/S架构，那来看看是怎么实现的

示意图
  • Client进程：使用服务的进程 —— Android客户端
  • Server进程：提供服务的进程 —— 服务端
  • ServiceManger：管理Service注册于查询 —— 类似路由功能
  • Binder驱动：
    1. 一种虚拟设备驱动，连接Server进程、Client进程和ServiceManger的桥梁；
    2. 传递服务进程消息；
    3. 通过内存映射传递进程间需要传递是数据；
    4. 通过Binder线程池，实现线程控制，并由Binder驱动自身进行管理
      —— Binder驱动持有每个Server进程的在内核空间中的Binder实体，并给Client进程提供Binder实体的引用，类似字典目录。
过程

参照上面的示意图

  1. 服务端调用Binder驱动，然后通过系统调用ServiceManger中注册服务，①③；
  2. 注册成功，并且ServiceManger持有当前服务的Binder实体的引用，③⑥；
  3. 客户端调用Binder驱动，然后通过系统调用ServiceManger中通过名字去查找对应的服务，②③；
  4. ServiceManger通过系统调用Binder驱动，给客户端返回所持有的Server端的Binder实体的引用，③⑤；
  5. 通过上述对Binder驱动的多次调用，实现了Client —— ServiceManager —— Server 的架构实现，即⑦⑧⑨。

在Android中的具体代码实现

在源码中，Binder是一个类，实现了IBinder接口；这里通过AIDL来实现;这里我们实现一个每隔5s就向服务端新增一本书，然后再在客户端打印出来。

IBookManager.aidl 是定义的一个接口，里面的两个方法：getBookList用于从远程服务端获取图书列表，addBook用于往图书列表中添加一本书

第二步：远程服务端Service实现

先创建一个Service BookManagerService

public class BookManagerService extends Service {
private static final String TAG = "BookManagerService";
private AtomicBoolean mIsServiceDestoryed = new AtomicBoolean(false);
private RemoteCallbackList<IOnNewBookListener> mListeners = new RemoteCallbackList<>();

/**
 * 支持并发读写  自动的线程同步
 */
private CopyOnWriteArrayList<Book> mBookList = new CopyOnWriteArrayList<>();

private Binder mBinder = new IBookManager.Stub() {
    @Override
    public List<Book> getBookList() throws RemoteException {
        return mBookList;
    }

    @Override
    public void addBook(Book book) throws RemoteException {
        mBookList.add(book);
    }

    @Override
    public void registerListener(IOnNewBookListener listener) throws RemoteException {
        mListeners.register(listener);
    }

    @Override
    public void unRegisterListener(IOnNewBookListener listener) throws RemoteException {
        mListeners.unregister(listener);
        Log.i(TAG, "unRegisterListener: SUCCESS" );
    }

    @Override
    public void basicTypes(int anInt, long aLong, boolean aBoolean, float aFloat, double aDouble, String aString) throws RemoteException {

    }
};

@Override
public void onCreate() {
    super.onCreate();
    mBookList.add(new Book(1, "安卓开发艺术探索"));
    mBookList.add(new Book(2, "安卓源码设计模式解析与实践"));
    /**
     * 每隔5s加一本书 并通知下去
     */

    new Thread(new ServiceWorker()).start();
}

@Override
public void onDestroy() {
    mIsServiceDestoryed.set(true);
    super.onDestroy();
}

private void onNewBookArrived(Book book) throws RemoteException {
    mBookList.add(book);

    final int N = mListeners.beginBroadcast();
    Log.i(TAG, "onNewBookArrived: size" + N);

    for (int i = 0; i < N; i++) {
//   mListeners.getBroadcastItem(i).onNewBookArrived(book);

        IOnNewBookListener listener = mListeners.getBroadcastItem(i);
        if (listener != null) {
            listener.onNewBookArrived(book);
        }
    }
    mListeners.finishBroadcast();
}


private class ServiceWorker implements Runnable {

    @Override
    public void run() {
        while (!mIsServiceDestoryed.get()){
        try {
            Thread.sleep(5000);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }

        int bookId = mBookList.size() + 1;
        Book newBook = new Book(bookId, "new Book # " + bookId);
        try {
            onNewBookArrived(newBook);
        } catch (RemoteException e) {
            e.printStackTrace();
        }

    }
    }
}

@Nullable
@Override
public IBinder onBind(Intent intent) {
    return mBinder;
}}

第三步：客户端实现

客户端实现主要是绑定远程服务，然后将服务端返回的Binder对象转换成AIDL接口，然后就可以通过这个接口去调用服务端的远程方法

//onCreate中绑定服务
   Intent intentA = new Intent();
   intentA.setClass(this, BookManagerService.class);
   bindService(intentA, aidlConnection, Service.BIND_AUTO_CREATE);

//AIDL方式
private ServiceConnection aidlConnection = new ServiceConnection() {
    @Override
    public void onServiceConnected(ComponentName name, IBinder service) {
        mIBookManager = IBookManager.Stub.asInterface(service);
        try {
            List<Book> bookList = mIBookManager.getBookList();
            Log.i(TAG, "onServiceConnected: bookManager" + bookList.getClass().getCanonicalName());
            Log.i(TAG, "onServiceConnected: bookManager" + bookList.toString());
            mIBookManager.addBook(new Book(3, "第一行代码"));
            List<Book> newList = mIBookManager.getBookList();
            Log.i(TAG, "onServiceConnected: bookManager" + newList.toString());
            mIBookManager.registerListener(mIOnNewBookListener);
        } catch (RemoteException e) {
            e.printStackTrace();
        }
    }

    @Override
    public void onServiceDisconnected(ComponentName name) {

    }
};

最后不要忘了在onDestroy中解绑，绑定成功之后会通过bookManager去调用getBookList方法，才能回去图书信息；

过程分析

服务端

服务端首先创建一个service用来监听客户端的链接请求，然后创建一个AIDL文件，并将要暴露给客户端的接口在这个AIDL文件中声明，最后在service中实现这个AIDL；

客户端

客户端首先需要绑定服务端的service，绑定成功之后，将服务端返回的Binder对象转换成AIDL接口所属的类型，这样就能直接调用AIDL中的方法；
代码中 mIBookManager 就能直接调用 getBookList，addBook方法；

AIDL接口的创建

在创建AIDL接口的时候，我们创建了后缀名为aidl的文件，并声明了IBookManager接口 和addBook和 getBookList 这两个接口方法；
在AIDL中只有部分数据类型可用：

  • 基本数据类型
  • String和CharSequence
  • List：ArrayList，并且里面的每个元素都能够被AIDL支持
  • Map：HashMap,键值对每个元素都必须能被AIDL支持
  • Parcelable：实现了Parcelable接口对象
  • AIDL
    除了基本数据类型，其它类型的参数必须标上方向：in、out或inout，用于表示在跨进程通信中数据的流向；

远程服务端Service的实现

上面我们定义了服务端、客户端以及AIDL接口，那么我们看看AIDL接口是怎么实现的：
首先创建一个 BookManagerService 并继承Service，在onCreate方法中添加图书信息；
然后创建一个Binder对象，继承自IBookManager.Stub并实现它内部的AIDL方法；
最后需要在onBinder中返回这个Binder对象；当然，最后不要忘了在AndroidManifest中声明BookManagerService；
还有这里用到的是 CopyOnWriteArrayList 而不是List，是因为AIDL的方法运行是在Binder线程池中，当多个客户端访问时需要处理线程同步，而 CopyOnWriteArrayList 能自动线程同步；CopyOnWriteArrayList 并不是继承自ArrayList，但依然能在此处使用；因为AIDL中支持的是抽象List，虽然服务端返回的 CopyOnWriteArrayList ,但是再Binder中依然会按照List的规范访问并完成一个新的ArrayList传递给客户端。
