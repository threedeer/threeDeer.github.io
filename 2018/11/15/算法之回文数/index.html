算法之回文数

题目描述

判断一个整数是否是回文数。回文数是指正序（从左向右）和倒序（从右向左）读都是一样的整数。

示例 1:

输入: 121
输出: true
示例 2:

输入: -121
输出: false
解释: 从左向右读, 为 -121 。 从右向左读, 为 121- 。因此它不是一个回文数。
示例 3:

输入: 10
输出: false
解释: 从右向左读, 为 01 。因此它不是一个回文数。

进阶：
你能不将整数转为字符串来解决这个问题吗？

思路

上面的进阶提示，让不要用字符串来解决，那么我就先看看字符串怎么解决吧；

public boolean isPalindromeStr(int x) {
    String num=String.valueOf(x);
    return new StringBuffer(num).reverse().toString().equalsIgnoreCase(num);
}

可以看得出来，很简单，这个方法我开始是不知道的，百度百科里面抄来的。代码就是讲字符串反转再跟原始数据比较。

看一眼执行效率：

看样子懒方法不一定有好的效果。

下面说说我的想法，所谓回文数就是正反一样，那么把数字拆开再反转必然是相等的，结合我的另一篇的 反转整数，就很容易解决了。

代码实现

 public boolean isPalindrome(int x) {
    if (x < 0) {
        return false;
    }
    int outNum = 0; 
    int m = 0; // 余数
    String str = String.valueOf(x);
    int length = str.length() / 2;
    int before = 0;
    int back = 0;
    if (str.length() % 2 != 0) {
        length += 1;
    }

    for (int i = 0; i < length; i++) {
        m = x % 10;
        x = x / 10;
        if (str.length() % 2 == 0) {
            //偶数
            before = m + outNum * 10;
        } else {
            //奇数
            if (i != length - 1) {
                //不最后一个
                before = m + outNum * 10;
            }
        }
        back = x;
        outNum = m + outNum * 10;
    }

    if (before == back) {
        return true;
    } else {
        return false;
    }

}
